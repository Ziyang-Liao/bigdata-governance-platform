# 大数据治理平台 — 功能差距分析与详细设计文档

> 版本: v2.0 | 日期: 2026-03-29
> 对标产品: 阿里 DataWorks、腾讯 WeData、Apache SeaTunnel、Apache NiFi、Databricks Unity Catalog

---

## 一、现状总结

当前平台已实现基础框架，但距离商用级产品在**自动化程度、交互细节、运维能力**上存在显著差距。以下按模块逐一分析。

---

## 二、模块级差距分析

### 模块 1：数据源管理

#### 现有功能
- 手动填写 host/port/user/password 创建数据源
- 测试连接（调 Glue Connection API）
- 浏览表/字段元数据

#### 缺失功能（共 18 项）

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 1.1 | **自动创建 Glue Connection** | P0 | 用户保存数据源时，后端自动创建 Glue Connection + 安全组 + 子网配置，用户无需手动操作 |
| 1.2 | **自动创建 IAM Role** | P0 | 后端自动创建/复用 Glue ETL 所需的 IAM Role，附加最小权限策略 |
| 1.3 | **自动配置网络（VPC/子网/安全组）** | P0 | 检测 RDS 所在 VPC，自动选择子网、创建安全组规则（允许 Glue → RDS 3306/5432 等端口） |
| 1.4 | **密码加密存储（Secrets Manager）** | P0 | 密码不存 DynamoDB，自动创建 Secrets Manager Secret，DynamoDB 只存 ARN |
| 1.5 | **连接池 & 连接状态心跳** | P1 | 定期检测数据源可达性，自动更新状态（active/unreachable/error） |
| 1.6 | **RDS 实例自动发现** | P1 | 调 RDS DescribeDBInstances API，列出账号内所有 RDS 实例供用户选择，免手动填写 host |
| 1.7 | **更多数据源类型** | P1 | 支持 Aurora、DynamoDB、MongoDB、Kafka、S3 (CSV/JSON/Parquet)、Redshift（作为源） |
| 1.8 | **SSL/TLS 连接配置** | P1 | 支持配置 SSL 证书、强制加密连接 |
| 1.9 | **连接参数高级配置** | P2 | JDBC 参数（charset、timezone、fetchSize 等）、连接超时、读超时 |
| 1.10 | **数据源分组/标签** | P2 | 按环境（dev/staging/prod）、业务线分组管理 |
| 1.11 | **数据源权限控制** | P1 | 不同用户/角色只能看到授权的数据源 |
| 1.12 | **数据预览** | P1 | 在元数据浏览中支持 SELECT TOP 10 预览数据 |
| 1.13 | **表行数/大小统计** | P2 | 显示每张表的行数、数据大小、最后更新时间 |
| 1.14 | **Schema 变更检测** | P2 | 定期对比源表 Schema，检测字段新增/删除/类型变更，通知用户 |
| 1.15 | **批量导入数据源** | P2 | 支持 CSV/JSON 批量导入数据源配置 |
| 1.16 | **连接测试详细报告** | P1 | 测试连接时返回详细信息：网络连通性、认证结果、权限检查（SELECT/SHOW DATABASES 等） |
| 1.17 | **数据源使用统计** | P2 | 显示该数据源被多少同步任务引用、最近同步时间 |
| 1.18 | **操作审计日志** | P1 | 记录谁在什么时间创建/修改/删除了数据源 |

### 模块 2：数据同步

#### 现有功能
- 5 步向导（基本配置 → 选表 → 字段映射 → 目标配置 → 调度）
- 支持 Glue/Zero-ETL/DMS 三种通道
- S3 分区配置、Redshift DISTKEY/SORTKEY
- 启停控制

#### 缺失功能（共 22 项）

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 2.1 | **通道自动推荐** | P0 | 根据源类型+目标类型+同步模式，自动推荐最优通道（Zero-ETL > Glue > DMS），并说明原因 |
| 2.2 | **字段类型自动映射** | P0 | MySQL INT → Redshift INTEGER、VARCHAR(255) → VARCHAR(255) 等，自动转换并标注不兼容类型 |
| 2.3 | **字段转换函数** | P1 | 支持在字段映射中添加转换：类型转换、脱敏（手机号/邮箱）、默认值、表达式计算 |
| 2.4 | **数据过滤条件（WHERE）** | P0 | 支持配置 WHERE 条件过滤源数据，如 `created_at > '2026-01-01'` |
| 2.5 | **增量字段配置** | P0 | 增量同步时指定增量字段（如 updated_at）、增量起始值、水位线管理 |
| 2.6 | **目标表自动建表** | P0 | 根据源表 Schema + 字段映射，自动在 Redshift/S3 生成 CREATE TABLE DDL 并执行 |
| 2.7 | **目标表名映射** | P1 | 支持自定义目标表名（默认同名），支持添加前缀/后缀 |
| 2.8 | **多表批量配置** | P1 | 选择多张表后，支持批量设置相同的目标配置、调度策略 |
| 2.9 | **数据量预估** | P2 | 同步前预估数据量（行数、大小），帮助用户评估成本和时间 |
| 2.10 | **同步速率限制** | P1 | 配置最大并发数、每秒写入行数限制，避免压垮源库 |
| 2.11 | **错误处理策略** | P1 | 脏数据处理：跳过/记录到错误表/中止任务，可配置容错行数 |
| 2.12 | **运行历史 & 统计** | P0 | 每次运行的开始/结束时间、读取行数、写入行数、耗时、状态、错误信息 |
| 2.13 | **任务详情页** | P0 | 独立的任务详情页面：配置信息、运行历史、日志、统计图表 |
| 2.14 | **任务克隆** | P2 | 一键复制现有任务配置，修改后创建新任务 |
| 2.15 | **任务版本管理** | P2 | 配置变更历史，支持回滚到历史版本 |
| 2.16 | **依赖关系** | P1 | 任务间依赖：A 完成后自动触发 B |
| 2.17 | **通知配置** | P1 | 任务成功/失败时发送通知（邮件/SNS/企业微信/钉钉 Webhook） |
| 2.18 | **Glue Job 参数透传** | P1 | 高级用户可配置 Glue Job 参数：Worker 数量、Worker 类型、超时、Spark 参数 |
| 2.19 | **S3 Table Bucket 选择** | P1 | 下拉列出账号内 S3 Bucket，而非手动输入 |
| 2.20 | **Redshift Workgroup/Database 下拉** | P0 | 同步目标配置中，Workgroup/Database/Schema 从 API 动态获取，非手动输入 |
| 2.21 | **数据质量校验** | P2 | 同步后自动校验：源端行数 vs 目标行数、主键唯一性、非空检查 |
| 2.22 | **断点续传** | P2 | 增量同步失败后，从上次水位线继续，不重复同步 |

### 模块 3：ETL 编排（工作流）

#### 现有功能
- ReactFlow DAG 编辑器（3 种节点：同步/SQL/Python）
- 节点配置面板（Drawer）
- DAG → Airflow DAG 文件生成
- 发布到 S3 / 触发运行

#### 缺失功能（共 15 项）

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|
| 3.1 | **更多节点类型** | P1 | 条件分支节点、并行网关、等待节点、HTTP 请求节点、通知节点、子工作流节点 |
| 3.2 | **节点参数化** | P1 | 支持全局变量（如 ${run_date}）、节点间参数传递 |
| 3.3 | **节点运行状态可视化** | P0 | DAG 图上实时显示每个节点的运行状态（等待/运行中/成功/失败），颜色区分 |
| 3.4 | **节点日志查看** | P0 | 点击节点直接查看该节点的运行日志 |
| 3.5 | **DAG 校验** | P1 | 保存前校验：是否有环、是否有孤立节点、必填参数是否完整 |
| 3.6 | **DAG 版本管理** | P1 | 每次发布生成版本号，支持回滚到历史版本 |
| 3.7 | **运行历史列表** | P0 | 工作流运行历史：每次运行的状态、耗时、各节点执行情况 |
| 3.8 | **手动重跑失败节点** | P1 | 工作流部分失败时，支持从失败节点重跑，不需要全部重跑 |
| 3.9 | **节点拖拽面板** | P1 | 左侧节点面板，拖拽到画布创建节点（而非下拉菜单） |
| 3.10 | **画布缩放/全屏** | P2 | 支持画布缩放、全屏编辑、自动布局 |
| 3.11 | **工作流模板** | P2 | 预置常用模板：MySQL → S3 → Redshift、日报生成等 |
| 3.12 | **SLA 配置** | P2 | 配置工作流 SLA（如必须在 6:00 前完成），超时告警 |
| 3.13 | **工作流标签/分组** | P2 | 按业务线、环境分组管理 |
| 3.14 | **并发控制** | P1 | 配置最大并发运行数，避免资源争抢 |
| 3.15 | **工作流导入/导出** | P2 | JSON 格式导入导出工作流定义 |
