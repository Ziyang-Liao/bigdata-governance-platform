# SCH-05 补数据 (Backfill)

> 优先级: P1 | 模块: 调度管理

## 1. 功能概述
指定时间范围重新执行历史调度。选择日期范围后批量生成补数据任务，按时间顺序执行。用于数据修复、历史数据重新加载。前端：补数据对话框，选择任务+日期范围+并发数。进度条显示补数据进度。后端：根据日期范围生成多个运行实例，每个实例的 ${run_date} 变量设为对应日期。使用队列控制并发（最多 N 个同时运行）。Airflow: 调 POST /dags/{dagId}/dagRuns 创建多个 DAG Run，设置 execution_date。

## 2. 用户故事
