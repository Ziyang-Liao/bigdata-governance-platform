# DS-10 数据源分组与标签

> 优先级: P2 | 模块: 数据源管理

## 1. 功能概述
标签系统支持：环境标签（dev/staging/prod）、业务线标签、自定义标签。列表页按标签过滤和搜索，支持标签统计视图。

## 2. 用户故事
- 作为团队负责人，我管理 50+ 个数据源，希望按环境和业务线分组，快速定位目标数据源。
- 作为运维人员，我希望一键筛选所有生产环境数据源，检查健康状态。

## 3. 交互设计
```
数据源列表页顶部过滤栏:
┌──────────────────────────────────────────────┐
│ 环境: [全部▼] [dev] [staging] [prod]          │
│ 业务线: [全部▼] [电商] [支付] [用户中心]       │
│ 标签: [核心] [只读] [+]    🔍 搜索数据源名称   │
└──────────────────────────────────────────────┘

数据源表单中:
┌──────────────────────────────────────────────┐
│ 环境:     [prod ▼]                            │
│ 业务线:   [电商 ▼]                            │
│ 自定义标签: [核心] [主库] [✕]  [+添加标签]     │
└──────────────────────────────────────────────┘

列表中标签展示:
│ 业务主库-MySQL  [prod] [电商] [核心]  ● 已连接 │
```

## 4. API 设计
```
GET /api/datasources?env=prod&biz=ecommerce&tags=核心
Response: 过滤后的数据源列表

GET /api/datasources/tag-stats
Response: {
  env: { dev: 5, staging: 3, prod: 12 },
  biz: { ecommerce: 8, payment: 4, user: 3 },
  custom: { 核心: 6, 只读: 4, 主库: 3 }
}
```

## 5. 数据模型
bgp-datasources 新增:
- env: String (dev/staging/prod)
- biz: String (业务线)
- tags: List<String> (自定义标签)
GSI: env-index (PK: env, SK: updatedAt) 按环境快速查询

## 6. 后端实现方案
```
1. 创建/更新时保存 env, biz, tags 到 DynamoDB
2. 列表查询:
   - 有 env 过滤: 使用 GSI env-index Query
   - 有 biz/tags 过滤: FilterExpression
   - 组合过滤: GSI Query + FilterExpression
3. 标签统计: Scan 全表聚合 (数据量小时可接受，大时用缓存)
4. 自定义标签自动补全: 从历史标签中提取去重列表
```

## 7. AWS 服务依赖
- DynamoDB (GSI)

## 8. 安全考虑
- 标签值长度限制 50 字符，单个数据源最多 10 个自定义标签
- 标签不包含敏感信息（密码、密钥等）
- 环境标签影响权限：prod 环境数据源需要更高权限才能修改

## 9. 验收标准
- [ ] 创建数据源时可选择环境和业务线
- [ ] 支持自定义标签添加/删除
- [ ] 列表页可按环境、业务线、标签过滤
- [ ] 标签在列表中以彩色 Tag 展示
- [ ] 标签统计视图显示各标签数量
- [ ] 自定义标签支持自动补全
