# SYNC-10 任务详情页

> 优先级: P0 | 模块: 数据同步

## 1. 功能概述
独立的任务详情页面 /sync/{taskId}，包含：配置摘要（源/目标/通道/模式）、运行历史表格、字段映射视图、日志查看器、统计图表（行数/耗时趋势）。替代当前仅有列表+弹窗的简单交互。

## 2. 用户故事
- 作为数据开发者，我希望在一个页面上看到同步任务的全部信息，包括配置、运行历史、日志，不需要在多个弹窗间切换。

## 3. 交互设计
```
/sync/{taskId} 页面布局:
┌──────────────────────────────────────────────────────┐
│ ← 返回列表   用户表全量同步        [编辑] [启动] [删除] │
├──────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────┐  │
│ │ 通道: Glue ETL | 模式: 全量/覆盖 | 状态: ● 运行中│  │
│ │ 源: MySQL ecommerce.users (8字段)               │  │
│ │ 目标: S3 parquet + Redshift public.ods_users    │  │
│ │ 调度: 0 2 * * * (每天凌晨2点) 下次: 03-30 02:00 │  │
│ │ Glue Job: bgp-sync-ecommerce | Workers: 2×G.1X │  │
│ └─────────────────────────────────────────────────┘  │
├──────────────────────────────────────────────────────┤
│ [运行历史] [配置详情] [字段映射] [日志] [统计]        │
├──────────────────────────────────────────────────────┤
│ (Tab 内容区域)                                        │
│ 运行历史: 表格 + 趋势图                               │
│ 配置详情: 源端/目标端/过滤条件/增量配置完整展示        │
│ 字段映射: 源字段→转换→目标字段 完整映射表              │
│ 日志: 最近一次运行的 CloudWatch 日志                   │
│ 统计: 行数趋势/耗时趋势/成功率/数据量                  │
└──────────────────────────────────────────────────────┘
```

## 4. API 设计
```
GET /api/sync/{id} — 任务完整配置
GET /api/sync/{id}/runs — 运行历史 (复用 SYNC-09)
GET /api/sync/{id}/logs — 最近运行日志
GET /api/sync/{id}/stats — 统计数据 (成功率/平均耗时/数据量趋势)
```

## 5. 数据模型
复用现有 bgp-sync-tasks + bgp-task-runs。

## 6. 后端实现方案
```
1. 新增前端路由: /sync/[id]/page.tsx
2. 页面加载时并行请求: 任务配置 + 运行历史 + 统计
3. 日志 Tab 懒加载 (点击时才请求)
4. 统计数据从 bgp-task-runs 聚合计算
5. 配置详情以只读 Descriptions 组件展示
6. 字段映射以表格展示，含类型映射和转换函数
```

## 7. AWS 服务依赖
- DynamoDB, CloudWatch Logs, Glue

## 8. 安全考虑
- 密码字段在配置详情中脱敏显示
- 日志中敏感信息过滤
